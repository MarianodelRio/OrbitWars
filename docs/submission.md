# Kaggle Submission

## Packaging a Bot

### Heuristic bot

Edit `scripts/submission/config.json`:

```json
{
  "bot": "bots.heuristic.sniper:agent_fn",
  "message": "sniper v2",
  "competition": "orbit-wars",
  "save_log": true
}
```

Then run:

```bash
python scripts/submission/run.py
```

### Neural bot

Edit `scripts/submission/neural_config.json`:

```json
{
  "checkpoint_path": "runs/my_run/run_001/checkpoints/best.pt",
  "message": "neural v3 il+rl",
  "competition": "orbit-wars",
  "save_log": true
}
```

Then run:

```bash
python scripts/submission/package_neural.py
```

Both scripts write `submission/main.py` and then submit it via the `kaggle` CLI.

## How Packaging Works

### Heuristic bots (`scripts/submission/run.py`)

1. The bot spec `module.path:attr` is imported and the attribute is inspected.
2. If `attr` is a plain function, its source is extracted and renamed to `agent()`.
3. If `attr` is a class instance, all relative-import dependencies are collected (topological order), their internal imports are stripped, and the class source is inlined. A `_bot = ClassName()` singleton and `def agent(obs, config=None)` wrapper are appended.
4. The result is written to `submission/main.py` with a standard header importing `math`.

### Neural bot (`scripts/submission/package_neural.py`)

Source files are concatenated in dependency order:

```
types.py → state_builder.py → action_codec.py → planet_policy_model.py → policy_sampler.py → bot.py
```

For each file:
- Relative imports (`from .`, `from bots.`, `from dataset.`) are stripped.
- Absolute stdlib/third-party imports are deduplicated.
- `from __future__ import annotations` is placed once at the top.
- `class NeuralBot(Bot):` is rewritten to `class NeuralBot:` (removes the base class dependency).

If `checkpoint_path` is provided and the file exists:
- The checkpoint `.pt` file is base64-encoded and embedded as a string literal `_CHECKPOINT_B64`.
- The generated `agent()` function decodes and loads the checkpoint on first call using `torch.load(io.BytesIO(...))`.

If no checkpoint is provided (or the file is not found), `agent()` initialises random weights.

## Submission Command

Both scripts invoke the Kaggle CLI automatically after writing `submission/main.py`:

```bash
kaggle competitions submit orbit-wars -f submission/main.py -m "your message"
```

You can also submit manually after packaging:

```bash
python scripts/submission/package_neural.py  # writes submission/main.py without submitting
kaggle competitions submit orbit-wars -f submission/main.py -m "test"
```

## Troubleshooting

- **`KAGGLE_API_TOKEN` not set**: Export your token or place `kaggle.json` in `~/.kaggle/`. See the [Kaggle API docs](https://www.kaggle.com/docs/api).
- **`checkpoint not found` warning**: The path in `neural_config.json` is resolved relative to the repo root. Check the path exists before packaging.
- **`torch not available` in submission**: The neural packaging embeds all model code inline. If the Kaggle environment does not have PyTorch, the submission will fail at runtime. Verify the competition environment supports PyTorch.
- **Submission file too large**: Base64-encoded checkpoints increase file size. Large models may exceed Kaggle's 50 MB submission limit. Use a smaller architecture or quantize before submission.

## Neural Bot Submission

`scripts/submission/package_neural.py` produces a fully self-contained `submission/main.py` with no imports from `bots/`, `game/`, or `dataset/`. The file structure after packaging:

```python
"""
Kaggle submission entry point — auto-generated by scripts/submission/package_neural.py.
Do not edit manually.
"""
from __future__ import annotations
import math
import torch
# ... other stdlib/third-party imports ...

# === types.py ===
# (ActionContext, PerPlanetLabels dataclasses)

# === state_builder.py ===
# (StateBuilder class)

# === action_codec.py ===
# (ActionCodec class, BINS)

# === planet_policy_model.py ===
# (PlanetPolicyConfig, PlanetPolicyOutput, PlanetBlock, PlanetPolicyModel)

# === policy_sampler.py ===
# (PolicySampler)

# === bot.py ===
# (NeuralBot class, stripped of Bot base class)

# === CHECKPOINT EMBEDDING ===
_CHECKPOINT_B64 = """..."""   # base64-encoded .pt file (if checkpoint provided)

_bot = None

def agent(obs, config=None):
    global _bot
    if _bot is None:
        # decode and load checkpoint on first call
        ...
    return _bot.act(obs, config)
```

The Kaggle environment calls `agent(obs, config)` each turn. The checkpoint is decoded from the base64 string and loaded with `torch.load(io.BytesIO(...), map_location="cpu")` on the very first turn, then reused for all subsequent turns.
